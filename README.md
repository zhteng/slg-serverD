# SLG-Server 游戏后端

## 项目简介

是一个基于 **Go 语言** 的游戏服务器框架，采用分层架构与微服务设计思想，实现了玩家系统、建筑升级、部队训练、行军采集、军团管理、地图格子、跨服战斗等核心玩法。项目重点解决 **高并发数据一致性、分布式延迟队列、实时推送与多区服管理** 等关键技术点。

该框架可作为后端的原型或初期骨架，支持快速扩展和定制化开发。

---

## 技术栈

- **语言**：Go 1.21+
- **Web 框架**：Gin
- **数据库**：MySQL (持久化)
- **缓存/消息队列**：Redis (分布式锁、延迟队列 ZSET、Stream、Pub/Sub)
- **认证**：JWT (双 Token、bcrypt 密码哈希、内部服务鉴权)
- **实时通信**：WebSocket (Hub 模式，心跳保活)
- **微服务通信**：gRPC + Protobuf (跨服服务、战斗服务预留)
- **配置**：YAML + 环境变量
- **依赖注入**：手动 DI 容器
- **定时任务**：Redis ZSET + 动态休眠 + 唤醒通道

---

## 架构设计
**多区服架构**：
- 每个区服（王国）是一个独立进程，通过 `SERVER_ID` 环境变量区分。

- **地图服务**：
- 600x600 格子，划分为 15x15 分片，每分片 40x40，独立加锁。
- 热点地块 Redis 常驻，冷数据懒加载。
- 地块状态变更异步批量写库

- 注册/登录（bcrypt 密码哈希）
- JWT 双 Token（Access/Refresh），支持黑名单与刷新滚动
- 玩家基础信息、资源（金币、木材、煤矿等）
- 离线资源产出（惰性结算 + 定时扫描）

- **军团联盟**：
- 创建、加入（自由/申请）、踢人（事务+分布式锁）
- 成员职位管理（盟主、副盟主）

- **实时推送**：
- WebSocket Hub 模式，维护在线玩家映射
- 建筑完成、训练完成、行军到达等事件主动推送
- 客户端心跳保活（5秒超时自动断线）
